# Skill/Tool 教程 00：为什么 Agent 需要工具？怎么和 RAG 配合？

模型擅长语言与推理，但不擅长“实时执行”和“可靠取数”。工具（Tool/Skill）就是把外部能力封装给模型用。

## 1. Tool/Skill 能解决什么
- **取数**：查数据库、查接口、读文件
- **执行**：跑脚本、跑测试、生成报告
- **写入**：提交变更、创建工单（这类要权限与审计）

如果没有工具，模型回答很多时候只能“描述怎么做”，不能“真的做”。

## 2. Tool 与 RAG 的分工
- RAG：解决“知识来自哪里”（文档、规范、历史记录）
- Tool：解决“动作怎么执行”（查询、计算、变更、验证）

常见组合：
1) RAG 先检索到规范/规则
2) Tool 执行操作（比如生成文件、跑检查）
3) Tool 返回结果
4) 再根据结果输出报告，并引用资料与日志

## 3. 本仓库的落地建议
你现在的最小 demo（`demos/rag_min/`）先把“检索+引用”跑通。
下一阶段如果你要做 Agent，可以增加：
- `tools/`：放工具定义与 schema（即使是伪代码也行）
- `workflows/`：放“Prompt + RAG + Tool”组合的流程模板
